The core difference is that Git is the software tool used to manage code history locally, 
while GitHub and Bitbucket are online platforms where you host and share that code with others.



These are common Git commands used in various situations:

start a working area (see also: git help tutorial)
   clone      Clone a repository into a new directory
   init       Create an empty Git repository or reinitialize an existing one

work on the current change (see also: git help everyday)
   add        Add file contents to the index
   mv         Move or rename a file, a directory, or a symlink
   restore    Restore working tree files
   rm         Remove files from the working tree and from the index

examine the history and state (see also: git help revisions)
   bisect     Use binary search to find the commit that introduced a bug
   diff       Show changes between commits, commit and working tree, etc
   grep       Print lines matching a pattern
   log        Show commit logs
   show       Show various types of objects
   status     Show the working tree status

grow, mark and tweak your common history
   backfill   Download missing objects in a partial clone
   branch     List, create, or delete branches
   commit     Record changes to the repository
   history    EXPERIMENTAL: Rewrite history
   merge      Join two or more development histories together
   rebase     Reapply commits on top of another base tip
   reset      Set `HEAD` or the index to a known state
   switch     Switch branches
   tag        Create, list, delete or verify tags

collaborate (see also: git help workflows)
   fetch      Download objects and refs from another repository
   pull       Fetch from and integrate with another repository or a local branch
   push       Update remote refs along with associated objects

-----------------------------------------------------------
git reset--used when commit is completed.


soft reset----remove commit history and keeps changes in staging area

git log --online
git reset --soft HEAD~1-----------modified green
git restore --staged--------------modified red
git restore-----------------------previous commit
------------------------------------------------------------
mixed reset----remove commit history and unstage the file

git log --online
git reset --mixed HEAD~1--------modified red
git restore---------------------previous commit
------------------------------------------------------------
Hard reset---remove commit history and directly to previous commit

git reset --hard HEAD~1----------directly to previous commit

git log

A pipeline is a concept, not a file, the automated CI/CD process from code to deployment.
A pipeline is defined in a workflow file/Jenkinsfile.

So, pipeline is the overall automation process, while workflow (GitHub Actions) and Jenkinsfile (Jenkins) are the files that define that process in their respective tools.

A DevOps pipeline is an automated workflow that moves code from a developer's machine into production. 
By automating the building, testing, and deployment of software, pipelines eliminate manual hand-offs, reduce human error
and allow teams to deliver high-quality updates continuously and reliably
